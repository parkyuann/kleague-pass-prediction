⚽ K-League Pass Prediction Improvement Project
"겁쟁이 모델(Under-prediction)을 과감한 스트라이커로 만들기"
데이터 분석과 수학적 보정(Calibration)을 통해 롱패스 예측 성능을 획기적으로 개선한 프로젝트입니다.
📌 1. 프로젝트 개요 (Overview)
	•	주제: K리그 선수들의 패스 성공 여부 및 도착 좌표 예측 모델 성능 개선
	•	문제 상황:
	◦	기존 모델(Classifier + Regressor)이 30m 이상의 롱패스를 예측할 때, 거리를 실제보다 20m 이상 짧게 예측하는 치명적인 문제 발생.
	◦	전체적인 RMSE는 나쁘지 않으나, 결정적인 롱패스 상황에서의 신뢰도가 매우 낮음.
	•	핵심 원인:
	◦	평균 회귀 (Regression to the Mean): 데이터의 대다수(90%)를 차지하는 숏패스(10~15m)에 편향되어, 롱패스 예측 시 안전한 평균값으로 수렴하려는 경향.
	◦	데이터 불균형: 롱패스 데이터 부족으로 인한 학습 미흡.
🕵️‍♀️ 2. 문제 진단 (Diagnosis)
모델의 예측 패턴을 분석하기 위해 다양한 시각화 기법을 도입했습니다.
2.1. 데이터 시각화 분석
	1	화살표 시각화 (Arrow Plot):
	◦	현상: 실제 패스(파란색)는 길게 뻗어나가는데, 예측 패스(빨간색)는 중간에 뚝 끊기는 현상 발견.
	◦	의미: 방향(Angle)은 정확하나 거리(Distance) 예측에 실패함.
	2	에러 히트맵 (Error Heatmap):
	◦	현상: 에러가 경기장 전체가 아닌, **수비 진영(Defense Zone)**과 측면에 집중됨.
	◦	의미: 수비수가 전방으로 길게 차는 '빌드업 롱패스'가 주된 에러 원인임.
	3	거리 산점도 (Scatter Plot):
	◦	현상: 실제 거리가 늘어나도 예측 거리는 25~30m 구간에서 천장(Ceiling)에 막힌 듯 더 이상 증가하지 않음.
2.2. 팀 전술 분석 (Clustering)
팀마다 패스 스타일이 다를 것이라는 가설 하에, K-Means 클러스터링을 통해 팀을 분류하고 특성을 분석했습니다.
	•	분석 변수: 1. 평균 패스 시작 위치 (Line Height): 수비 라인을 얼마나 올리고 경기하는가? (start_x) 2. 평균 패스 거리 (Pass Length): 롱볼 위주인가, 숏패스 위주인가? (avg_pass_dist)
	•	3분류 (3-Clusters) - 기본 스타일:
	◦	Group A (수비적 롱볼): 라인을 내리고(낮은 start_x) 긴 패스를 주로 구사하는 팀 (역습형).
	◦	Group B (밸런스): 평균적인 위치와 거리 분포를 보이는 팀.
	◦	Group C (공격적 숏패스): 라인을 올리고(높은 start_x) 짧은 패스로 점유율을 가져가는 팀 (지배형).
	•	5분류 (5-Clusters) - 심화 전술:
	◦	3분류로는 설명되지 않는 미세한 전술 차이를 반영하기 위해 그룹을 세분화했습니다.
	◦	Cluster 0 (극단적 텐백): 라인을 페널티 박스 근처까지 완전히 내리고 롱볼만 노리는 팀.
	◦	Cluster 1 (안전한 점유): 라인은 높지만 패스 거리가 매우 짧은(Tiki-taka) 팀.
	◦	Cluster 2 (빌드업 중시): 수비 지역에서 시작하지만 짧은 패스로 풀어나가는 팀.
	◦	Cluster 3 (다이렉트 공격): 높은 위치에서 공을 탈취해 바로 롱패스를 찌르는 전방 압박 팀.
	◦	Cluster 4 (밸런스): 모든 지표가 평균적인 팀.
	◦	이 5가지 클러스터 ID를 모델의 피처(Feature)로 추가하여 팀별 맞춤 예측을 유도했습니다.
🛠️ 3. 해결 전략 (Solution Strategy)
단순한 재학습(Retraining)으로는 구조적인 편향을 해결하기 어렵다고 판단하여, "모델의 예측값을 수학적으로 교정하는 후처리(Post-processing)" 전략을 수립했습니다.
✅ 전략 1: 구역별(Zone) 맞춤형 보정 (Zonal Calibration)
	•	아이디어: 에러 패턴이 구역마다 다르므로, 경기장을 수비/중원/공격으로 나누어 별도의 보정 공식을 적용.
	•	방법: 각 구역의 롱패스 데이터만 추출하여 2차 함수(Polynomial Regression) 모델 학습.
	•	효과: 에러가 가장 컸던 수비 지역의 예측 거리를 효과적으로 늘려줌.
✅ 전략 2: 하이브리드 탐지 시스템 (Hybrid Detection)
	•	아이디어: 무조건 늘리면 숏패스까지 늘어나는 부작용 발생. "진짜 롱패스만 골라내자!"
	•	방법:
	◦	LightGBM 분류기: 위치 정보(start_x, y)와 예측 거리(pred_dist)를 입력받아 롱패스일 확률 예측.
	◦	조건부 적용: "LGBM이 롱패스라고 확신할 때" + "기존 모델이 25m 이상이라고 했을 때"만 보정 적용.
