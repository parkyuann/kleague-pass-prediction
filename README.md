**K-League 최종 패스 좌표 예측 프로젝트**
Dacon K-League 패스 좌표 예측 대회 > 데이터 기반의 패스 궤적 분석을 통해 최종 도달 좌표(x, y)를 예측하는 머신러닝 모델 구축 및 분석 프로젝트입니다.

**1. 프로젝트 요약**
목표: 선수 및 경기 이벤트 데이터를 활용한 패스 도달 지점 예측

성과: Dacon 대회 상위 30%(290/937) 달성

핵심 강점: 모델링에 그치지 않고, 예측 실패 사례(Hard Cases)를 시각화하여 데이터의 특성을 분석하고 개선 방향을 도출함.

**2. 모델 실험 및 비교 분석**
최적의 예측 성능을 도출하기 위해 정형 데이터에 강한 트리 모델과 시계열 특성을 반영한 딥러닝 모델 두 가지 경로로 실험을 진행했습니다.

📊 모델 성능 비교
| 비교 항목 | 트리 기반 앙상블(v8_pipeline) | 딥러닝 시퀀스 모델(v3) |
| 모델 구조 | LightGBM + XGBoost + CatBoost | GRU/LSTM 기반 Multi-task Model |
| 핵심 전략 | 정교한 피처 엔지니어링 및 과적합 제어 | 경기 흐름(이벤트 시퀀스) 학습 |
| 결과 | 최종 채택(높은 일반화 성능) | 성능 실험 및 분석 수행 |

**3. 주요 수행 내용**
🛠️ 데이터 파이프라인 및 모델링 (train_v8_pipeline.py)
앙상블 기법 적용: LightGBM, XGBoost, CatBoost 모델을 결합하여 단일 모델의 편향을 최소화.

과적합 완화 전략:

max_depth 감소 및 l2_leaf_reg 증가를 통한 정규화(Regularization) 강화.

변별력이 낮거나 노이즈가 심한 피처(prev1_dist, angle_to_goal_diff 등) 과감히 제거.

자동화 워크플로우: Claude Code 및 Cursor CLI를 활용하여 피처 생성부터 파라미터 튜닝까지 실험 사이클을 자동화.

🔍 에러 분석 및 시각화 (03_Hard_Cases_Visualization.ipynb)
단순 오차 수치 확인을 넘어, 실제 축구장 좌표 위에서 모델이 왜 틀렸는지를 분석했습니다.

(여기에 직접 캡처한 경기장 분석 이미지를 삽입하세요)

Hard Cases 분석 결과:

시작 위치에 따른 편차: 수비 지역 및 중앙 지역에서 시작되는 패스의 예측 난이도가 높음.

Under-prediction 현상: 20m 이상의 롱패스에서 실제 거리보다 짧게 예측하는 '평균 회귀' 경향 확인.

**4. 핵심 인사이트 및 한계점**
💡 인사이트
이벤트 기반 거리 예측: 'Carry'나 'Pass' 등 이벤트 타입별 평균 이동 거리가 예측의 강력한 기준점이 됨을 확인했습니다.

도메인 지식의 중요성: 축구의 전술적 특성상 전진 패스와 횡패스의 물리적 데이터 특성이 다름을 시각화로 증명했습니다.

⚠️ 프로젝트의 한계점
극단값 예측 미흡: 모델이 전체적인 평균을 맞추려다 보니, 결정적인 롱패스나 창의적인 킬패스 같은 아웃라이어(Outlier)를 충분히 포착하지 못함.

상황 정보 부족: 주변 수비수의 위치나 패스를 받는 선수의 속도 등 세부적인 맥락 데이터가 결여되어 예측의 정교함에 한계가 있음.

**5. 향후 개선 방향**
딥러닝 도입: 정형 데이터 기반의 트리 모델을 넘어, 패스 전후의 흐름을 학습할 수 있는 시계열 모델(RNN/LSTM) 도입 검토.

피처 엔지니어링 고도화: 단순 좌표차이 외에 '패스 각도 변화율', '수비 밀집도 추정치' 등 더 고차원적인 피처 설계.

손실 함수 커스터마이징: 롱패스의 오차에 가중치를 주는 가중치 손실 함수(Weighted Loss) 적용 시도.
